What is database?
It is a place where data is stored persistently.
Servers are transient , database is persistent.

Examples of data stored in databases
For Linkedin:
1. User data
2. User posts
3. Users connection relationships
4. Messages

Why can't we let the user hit the database directly?
Databases were created using protocols that browsers don't understand.
Databases don't have granual access as a first class citizen. Very hard to do user specific access in them.

So servers has complete access to the databases and not the user.

Databases usually allow access to 4 primitives
1. Create Data
2. Read Data
3. Update Data
4. Delete Data
Popularly known as CRUD

In mongoose, first you have to define the schema.

Schema means structure of the data.
Users -> 
  email
  password
  firstname
  city
  age

This sounds counter intuitive, since mongodb is schemaless.
That is true, but mongoose makes you define schema for things like autocompletions/validating data before it goes to DB to make sure you are doing things right. 
Schemaless DBs can be very dangerous, using schema in mongo makes it slightly less dangerous. 

Defining Schema 
const UserSchema = new mongoose.Schema({
  email : String,
  password : String, 
  purchasedCourses : [{
    type : mongoose.Schema.Types.ObjectId,
    ref : "Course" 
  }]
})

const CourseSchema = new mongoose.model({
  title : String,
  price : 5999
}) 

const User = mongoose.model("User",UserSchema)
const Course = mongoose.model("Course",CourseSchema)
