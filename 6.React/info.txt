Why do you need React?

For most static websites, you don't need it because static websites content does not get changed and remains same after loading once.

For dynamic websites, the React library makes it easier to do DOM manipulation.

People realised it's harder to do DOM manipulation the conventional way.
There were libraries that came into the picture that made it slightly easier, but still for a very big app it's very hard.
Eventually, VueJS/ReactJS created a new syntax to do frontends.
Under the hood, the react compiler converts your code to HTML/CSS/JS.

Some react jargon,
To create a react app, you usually need to worry about 2 things:
1. State
  An object that represents the state of the app. It represents the dynamic things in your app.  (things that change).

  For the counter app, it could look something like this:
  {
    count : 1
  }
  For the Linkedin Topbar, it could be something like this:
  {
    topbar : {
      home : 0,
      myNetwork : "99+",
      jobs : 0,
      messaging : 0,
      notifications : 10
    }
  }

2. Components
  How a DOM element should render, given a state. It is a reusable, dynamic, HTML snippet that changes given the state.
   
Re-rendering - A state change triggers re render. A re render represents the actual DOM being manipulated when the state changes.

You usually have to define all the componenets once. and then all you have to do is update the state of your app. React takes care of re-rendering your app.
